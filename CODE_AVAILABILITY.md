
# Code Availability (Ready-to-Paste)

## Generic (most journals)
The complete analysis pipeline used in this study (DVH preprocessing, dose metrics, NTCP modeling using LKB and Relative Seriality, machine‑learning modules, QA reporting, and clinical‑factor analyses) is available in a private GitHub repository and will be made public upon acceptance. During peer review, editors and reviewers may access the code upon request. A citable, archived release will be deposited to Zenodo with an embargoed DOI; the DOI will be activated at acceptance to ensure long‑term accessibility and reproducibility.

## Springer Nature (Code availability)
Code for DVH preprocessing, NTCP modeling (Lyman–Kutcher–Burman, Relative Seriality), machine‑learning analyses, and figure generation is available from the corresponding author upon reasonable request for peer review. Following acceptance, the code will be publicly released under an open‑source license with a Zenodo DOI.

## Elsevier (Code availability)
The software pipeline supporting the findings of this study is available from the corresponding author on reasonable request for the purposes of peer review. Upon article acceptance, the repository will be made public and archived on Zenodo with a DOI to ensure reproducibility.

## IEEE (Data, Materials, and Code Availability)
The software used to generate the results in this paper is available to editors and reviewers on request. Upon acceptance, we will release the repository publicly and archive a versioned snapshot on Zenodo with a DOI.

## PLOS (Data/Code Availability)
All code underlying the results reported in this manuscript will be made publicly available at the time of publication. For peer review, code access can be provided to editors and reviewers upon request. A DOI‑minted archive (Zenodo) will be provided upon acceptance.
